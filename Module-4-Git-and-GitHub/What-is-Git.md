# 🌐 Git Basics

## 📘 Introduction
**Git** is a free, open‑source **distributed version control system (DVCS)** created by Linus Torvalds in 2005.  
It is used to track changes in source code, manage project history, and enable collaboration among developers.

# 🔹 1. Why Git?
- Tracks changes in files over time.  
- Allows multiple developers to work on the same project.  
- Provides branching and merging for experimentation.  
- Ensures data integrity with checksums.  
- Works offline since every developer has a full copy of the repository.

---

# 🔹 2. Git vs Centralized VCS

| Feature              | Git (DVCS) | Centralized VCS (e.g., SVN) |
|-----------------------|------------|-----------------------------|
| Storage model         | Snapshots  | File differences (deltas)   |
| Speed                 | Very fast (local ops) | Slower (server dependent) |
| Collaboration         | Fully distributed | Requires central server |
| Branching             | Lightweight, easy | Heavy, slower |
| Offline work          | Possible | Not possible |
